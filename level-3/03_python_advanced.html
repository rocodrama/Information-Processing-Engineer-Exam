01 얕은 복사 vs 깊은 복사
얕은 복사(copy.copy, lst[:], list(), dict.copy())는 바깥 컨테이너만 새로 만들고 내부의 가변 객체(리스트/딕셔너리)는 원본과 그대로 같은 객체를 공유합니다.
완전히 독립된 복사본이 필요하면 copy.deepcopy()를 써야 내부까지 새 객체가 만들어집니다.
기출변형 다음 Python 코드의 출력 결과를 쓰시오. (참조 대입 vs 슬라이싱 복사)

c.append(5)

print(c)

c = a[:]는 슬라이싱으로 만든 새 리스트(얕은 복사)이므로 c를 바꿔도 a·b는 영향받지 않습니다.

02 가변 기본 인자 함정
함수의 기본 인자(default argument)는 def가 실행되는 시점에 단 한 번만 생성됩니다. 리스트/딕셔너리 같은 가변 객체를 기본값으로 쓰면, 인자를 생략하고 호출할 때마다 같은 객체가 재사용되어 값이 계속 누적됩니다.
기출변형 다음 Python 코드의 출력 결과를 쓰시오. (기본 리스트 인자 누적)
▼ 
def add_item(item, lst=[]):
    lst.append(item)
    return lst

print(add_item('a'))
print(add_item('b'))
print(add_item('c'))
호출기본값 lst(D, def 시점에 한 번 생성)반환값
add_item('a')D=[] → append('a') → D=['a']['a']
add_item('b')D=['a'] (재사용) → append('b') → D=['a','b']['a', 'b']
add_item('c')D=['a','b'] (재사용) → append('c') → D=['a','b','c']['a', 'b', 'c']
정답
['a']
['a', 'b']
['a', 'b', 'c']
lst=[]는 함수가 정의될 때 딱 한 번만 만들어지는 객체입니다. 인자를 생략한 채 호출할 때마다 이 동일한 리스트에 계속 append되므로 값이 매 호출마다 누적됩니다.

03 클로저 & 스코프
클로저는 변수의 값이 아니라 변수가 담긴 스코프(셀) 자체를 참조합니다. 반복문 안에서 만든 함수들은 나중에 호출되는 시점의 변수 값을 보게 되는 지연 바인딩(late binding) 함정에 주의해야 합니다.
기출변형 다음 Python 코드의 출력 결과를 쓰시오. (nonlocal과 클로저 상태 유지)

def counter():
    count = 0
    def increment():
        nonlocal count
        count += 1
        return count
    return increment

c1 = counter()
print(c1())
print(c1())
c2 = counter()
print(c2())
print(c1())

c1()1→2-2

c2()20→11

import re

log = "2024-07-15 ERROR Something failed"
pattern = r'(?P<date>\d{4}-\d{2}-\d{2}) (?P<level>\w+) (?P<msg>.+)'
m = re.match(pattern, log)

print(m.group('date'))
print(m.group('level'))
print(m.groups())
print(m.group(0) == log)

심화 다음 Python 코드의 출력 결과를 쓰시오. (그룹 캡처 findall + 중복 제거 + count 제한 백레퍼런스 치환)
▼ 
import re

log = "user=kim;user=lee;user=park;user=kim"
users = re.findall(r'user=(\w+)', log)

unique_first = []
for u in users:
    if u not in unique_first:
        unique_first.append(u)

summary = ", ".join(f"{i+1}:{name}" for i, name in enumerate(unique_first))

print(users)
print(unique_first)
print(summary)

replaced = re.sub(r'user=(\w+)', r'[\1]', log, count=2)
print(replaced)
실행내용
re.findall(r'user=(\w+)', log)그룹이 1개라 그룹 캡처값만 리스트로 반환 → ['kim','lee','park','kim']
unique_first 누적kim(추가), lee(추가), park(추가), kim(이미 있음→건너뜀) → ['kim','lee','park']
summaryenumerate로 1:kim, 2:lee, 3:park 생성 후 ", "로 결합
re.sub(패턴, r'[\1]', log, count=2)앞의 2개 매치("user=kim","user=lee")만 [kim],[lee]로 치환, 나머지는 그대로
정답
['kim', 'lee', 'park', 'kim']
['kim', 'lee', 'park']
1:kim, 2:lee, 3:park
[kim];[lee];user=park;user=kim
패턴에 캡처 그룹이 있으면 re.findall은 매치 전체가 아니라 그룹 값만 반환합니다. re.sub의 r'[\1]'은 첫 번째 그룹을 대괄호로 감싸 치환하는 백레퍼런스이며, count=2로 앞의 두 매치만 치환하고 나머지는 원본 그대로 둡니다.
