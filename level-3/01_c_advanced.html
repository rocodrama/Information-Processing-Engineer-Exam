01 이중 포인터 & 포인터 배열
💡 int **pp는 "포인터를 가리키는 포인터". **pp는 pp → p → 대상값까지 두 단계 역참조. 포인터 배열 int *arr[n]의 원소는 각각 다른 변수의 주소를 담을 수 있다.
기출변형  다음 C 코드의 출력 결과를 쓰시오.
▼ 
1#include <stdio.h>
2int main() {
3    int x = 10;
4    int *p = &x;
5    int **pp = &p;
6    **pp = 20;
7    printf("%d %d", x, *p);
8    return 0;
9}

720&x&p202020 20
정답

pp→p→x 순서로 이어지므로 **pp는 x 자체를 가리킨다. **pp = 20은 x를 20으로 바꾸고, 이후 x와 *p는 모두 20이 된다.

02 배열-포인터 연산 & 포인터 산술
💡 포인터 + n은 (자료형 크기 × n)바이트 이동. 두 포인터의 차(p2-p1)는 바이트가 아니라 "원소 개수"로 계산된다. 2차원 배열 포인터는 int (*p)[열개수] 형태로 행 단위 이동.
기출변형  다음 C 코드의 출력 결과를 쓰시오.
▼ 
1#include <stdio.h>
2int main() {
3    int arr[5] = {2, 4, 6, 8, 10};
4    int *p = arr;
5    printf("%d %d %d", *(p+1), *(p+3), *(arr+4));
6    return 0;
7}
표현식인덱스값
*(p+1)arr[1]4
*(p+3)arr[3]8
*(arr+4)arr[4]10
출력4 8 10
정답
4 8 10
arr = {2,4,6,8,10}. p는 arr[0]을 가리키므로 *(p+n)은 arr[n]과 동일하다. arr+4도 배열 이름에 대한 포인터 연산이므로 동일 규칙 적용.
기출변형  다음 C 코드의 출력 결과를 쓰시오. (포인터 역방향 순회)

2int main() {

5    int sum = 0;
6    while (p >= arr) {
7        sum += *p;
8        p--;
9    }
10    printf("%d", sum);
11    return 0;
12}

p는 배열 끝(arr+3)에서 시작하여 arr에 도달할 때까지 한 칸씩 감소하며 모든 원소를 더한다. 20+15+10+5 = 50.

03 구조체 & 공용체(union), 중첩/배열 구조체
💡 struct는 각 멤버가 독립된 메모리를 가지지만, union은 모든 멤버가 "같은 메모리"를 공유한다(가장 큰 멤버 크기만큼 확보). 리틀 엔디안(x86) 기준 하위 바이트가 낮은 주소에 저장된다.
기출변형  다음 C 코드의 출력 결과를 쓰시오. (구조체 배열)
▼ 
1#include <stdio.h>
2struct Item { char name[10]; int price; };
3int main() {
4    struct Item items[3] = {{"Pen",1000}, {"Book",5000}, {"Bag",20000}};
5    int total = 0;
6    for (int i = 0; i < 3; i++) total += items[i].price;
7    printf("%d", total);
8    return 0;
9}
iitems[i].nameitems[i].pricetotal

1"Book"50006000
2"Bag"2000026000
정답
26000
구조체 배열을 순회하며 price 멤버만 누적. 1000+5000+20000 = 26000.
기출변형  다음 C 코드의 출력 결과를 쓰시오. (중첩 구조체)

1#include <stdio.h>

9    return 0;
10}

중첩 구조체는 r.br.x처럼 점(.)을 연달아 사용해 접근한다. w=5-1=4, h=6-2=4, 넓이 w*h=16.

04 동적 메모리 (malloc/calloc/realloc/free)
💡 malloc은 초기화 안 된(쓰레기값) 메모리, calloc은 0으로 초기화된 메모리를 반환. realloc은 기존 내용을 보존한 채 크기를 재조정. free 이후에도 포인터 변수 자체는 남아있는데(dangling pointer), 이를 역참조하면 정의되지 않은 동작(UB)이다.

심화  다음 C 코드의 출력 결과를 쓰시오. (재귀 함수 속 static 변수 공유)
▼ 
1#include <stdio.h>
2int trace(int n) {
3    static int depth = 0;
4    depth++;
5    if (n <= 0) {
6        printf("%d ", depth);
7        return depth;
8    }
9    int r = trace(n - 1);
10    depth += 1;
11    return r;
12}
13int main() {
14    int result = trace(3);
15    printf("%d", result);
16    return 0;
17}
호출depth++ 후동작r (반환값)
trace(3)1n>0 → trace(2) 호출대기
trace(2)2n>0 → trace(1) 호출대기
trace(1)3n>0 → trace(0) 호출대기
trace(0)4n<=0 → printf("4 "), return 44 (출력됨)
trace(1) 복귀5 (depth+=1)r=4 그대로 반환4
trace(2) 복귀6 (depth+=1)r=4 그대로 반환4
trace(3) 복귀7 (depth+=1)r=4 그대로 반환 → result=44
main printf—result 출력4
정답
4 4
static depth는 재귀 호출 전체가 "단 하나"의 변수를 공유한다. trace(3)→trace(2)→trace(1)→trace(0)까지 내려가며 depth가 1→4로 증가하고, trace(0)에서 depth(4)를 출력한다. 반환값 r=4는 그대로 위로 전달되어 최종 result=4가 되고 main에서 다시 4를 출력한다(그 사이 depth는 계속 증가하지만 반환값 r에는 반영되지 않음).
