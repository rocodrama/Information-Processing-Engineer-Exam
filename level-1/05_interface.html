2 핵심 개념

인터페이스 설계서 구성요소 (자주 출제!)

항목설명
인터페이스 ID/명칭각 인터페이스를 구분하는 고유 식별자와 이름
송신 시스템 / 수신 시스템데이터를 보내는 쪽과 받는 쪽 시스템 정보
연동 데이터 항목송수신되는 데이터의 항목, 타입, 길이
통신 규약(프로토콜)HTTP, TCP/IP, FTP 등 통신에 사용하는 규약
연동 방식동기(Synchronous) / 비동기(Asynchronous)
연동 주기실시간, 배치(주기적) 등 발생 빈도
보안 적용 여부암호화, 인증 적용 여부 및 방식
예외처리 방안오류 발생 시 처리 절차와 응답 코드

동기 vs 비동기 방식

구분특징
동기(Synchronous)송신 후 응답을 기다렸다가 다음 처리 진행, 실시간성 높음, 처리 지연 시 대기
비동기(Asynchronous)송신 후 응답을 기다리지 않고 다음 처리 진행, 별도 콜백/큐로 결과 수신

인터페이스 데이터 표준

구분XMLJSON
구조태그 기반, 계층적키-값 기반, 경량
가독성/용량상대적으로 무거움가볍고 파싱 빠름
주 용도문서형 데이터, 레거시 연계웹/모바일 API 연계
데이터 표준화: 공통 코드, 표준 데이터 형식을 정의하여 송수신 시스템 간 데이터 해석 차이를 방지합니다.

HTTP 응답 코드 분류 (자주 출제!)

코드대의미
1xx정보성 응답 (요청을 받았으며 처리 중)
2xx성공 (요청이 정상적으로 처리됨)
3xx리다이렉션 (다른 URI로 이동 필요)
4xx클라이언트 오류 (잘못된 요청, 인증/권한 문제)
5xx서버 오류 (서버 내부 처리 실패)

오류 처리 보고서 구성

오류 발생 일시, 오류 코드/메시지, 발생 위치(인터페이스명), 오류 원인, 처리 내용 및 재처리 결과를 기록합니다.

인터페이스 보안 적용 요소

구분적용 기술
전송 구간 암호화SSL/TLS로 네트워크 전송 데이터 암호화
인증API Key, OAuth, 토큰 기반 인증

접근통제호출 가능한 시스템/IP 제한

인터페이스 구현 검증 도구 (자주 출제!)
