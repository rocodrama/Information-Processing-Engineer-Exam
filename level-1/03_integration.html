2 핵심 개념

연계 데이터 식별 및 표준화

항목설명
연계 대상 시스템데이터를 주고받는 송신/수신 시스템 식별
연계 데이터 항목송수신 데이터의 항목, 타입, 길이 정의
코드 매핑시스템 간 서로 다른 코드값을 표준 코드로 매핑
데이터 표준화데이터 형식을 통일하여 해석 차이를 방지

EAI(Enterprise Application Integration) 구축 유형 (자주 출제!)

유형특징
Point-to-Point시스템 간 1:1 직접 연결, 구현은 단순하나 시스템 증가 시 연계 복잡도 급증
Hub & Spoke중앙 허브(Hub)를 두고 모든 시스템이 허브를 통해 연계, 허브 장애 시 전체 영향
Message Bus(ESB)버스를 통한 표준 메시지 기반 연계, 확장성이 가장 우수
HybridHub&Spoke와 Message Bus를 혼합한 방식

ESB(Enterprise Service Bus)

서비스 중심의 미들웨어로 메시지 라우팅, 데이터 형식 변환, 프로토콜 중계 등을 담당하며 시스템 간 결합도를 낮춥니다.

REST vs SOAP

구분RESTSOAP
기반HTTP 메서드(GET/POST/PUT/DELETE)XML 기반 프로토콜
데이터 형식주로 JSON (경량)XML (WSDL로 명세 정의)
특징가볍고 빠름, 캐싱 용이보안·트랜잭션 기능이 강력하나 무거움

직접연계 vs 간접연계

구분직접연계간접연계
방식DB Link, API 직접 호출 등으로 시스템 간 직접 연결EAI/ESB 등 미들웨어를 경유하여 연계
결합도높음 (변경 시 영향범위 큼)낮음 (변경에 유연)
구축비용상대적으로 낮음상대적으로 높음 (미들웨어 도입 필요)

연계 보안 / 데이터 무결성

항목설명
연계 데이터 암호화전송 구간 SSL/TLS 등으로 데이터 보호
데이터 무결성 검증체크섬/해시값 비교로 데이터 위변조 여부 확인
접근통제연계 가능한 시스템/IP 제한

연계 주기 / 실패 처리

항목설명
연계 주기실시간(Real-time) 또는 배치(주기적, Batch) 처리
재처리(Retry)연계 실패 시 정해진 횟수만큼 재시도
오류 로그/알림실패 내역을 로그로 적재하고 담당자에게 알림
