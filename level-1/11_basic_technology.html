2 핵심 개념

프로세스 vs 스레드

구분프로세스 (Process)스레드 (Thread)
정의실행 중인 프로그램프로세스 내 실행 단위
자원독립적인 메모리 공간프로세스 자원 공유
통신IPC(파이프, 소켓 등)공유 메모리 직접 접근
생성 비용높음낮음 (경량 프로세스)
오류 영향독립적같은 프로세스 내 영향

교착상태(Deadlock) 4필요조건 (모두 충족해야 발생)

조건설명해결책
상호배제
Mutual Exclusion		자원은 한 번에 하나의 프로세스만 사용 가능자원 공유 허용
점유대기
Hold & Wait		자원을 점유한 채 다른 자원을 기다림자원 일괄 요청
비선점
No Preemption		타 프로세스가 자원을 강제로 빼앗을 수 없음강제 자원 회수
순환대기
Circular Wait		자원 요청이 원형(사이클)으로 대기자원 번호 순서 부여
암기: 상점비순 (상호배제 → 점유대기 → 비선점 → 순환대기)

메모리 관리 기법

기법설명특징
페이징메모리를 고정 크기 페이지로 분할외부 단편화 없음, 내부 단편화 발생
세그멘테이션논리적 단위(세그먼트)로 분할내부 단편화 없음, 외부 단편화 발생
가상 메모리보조기억장치를 RAM처럼 사용스왑(Swap), 페이지 폴트 발생
스래싱
Thrashing		페이지 교체가 너무 빈번해 성능 저하워킹셋(Working Set)으로 예방

프로세스 스케줄링 알고리즘 요약

알고리즘선점 여부특징단점
FCFS 비선점 도착 순서대로 처리 (선입선출) Convoy Effect
SJF 비선점 실행 시간 짧은 작업 먼저, 평균 대기시간 최소 기아(Starvation)
SRTF 선점 SJF의 선점형, 잔여시간 짧은 작업 실행 기아, 잦은 문맥교환
Priority 선점/비선점 우선순위 높은 작업 먼저 기아 → 에이징으로 해결
Round Robin 선점 시간 할당량(Time Quantum) 순환 TQ 크면 FCFS와 유사
HRN 비선점 SJF의 기아 보완
우선순위 = (대기+실행) / 실행		 계산 복잡
MLQ 선점 다단계 큐, 작업 유형별 큐 분리 큐 간 이동 불가
에이징(Aging): 오래 기다린 작업의 우선순위를 점진적으로 높여 기아 방지
HRN 공식: 우선순위 = (대기시간 + 실행시간) / 실행시간 → 값이 클수록 먼저

TCP vs UDP 핵심 비교 (자주 출제!)

구분TCPUDP
연결 방식연결 지향 (3-way handshake)비연결
신뢰성데이터 전달 보장, 순서 보장보장 없음

사용 예HTTP, FTP, 이메일DNS, 스트리밍, 게임
