블랙박스 테스트 기법

기법설명예시
동등분할
(Equivalence Partitioning)		 입력 데이터를 동등 클래스로 나눠 각 클래스에서 하나씩 테스트 1~100 유효 → 50 하나만 테스트
경계값 분석
(Boundary Value Analysis)		 유효·무효 경계의 값을 집중 테스트 (경계±1) 1~100이면 0,1,100,101 테스트
원인-결과 그래프
(Cause-Effect Graph)		 입력 조건(원인)과 출력 결과(결과)의 관계를 그래프로 표현 논리 관계 도출
결정 테이블
(Decision Table)		 조건과 행동을 표로 정리해 모든 조합 테스트 조건 조합표
상태 전이
(State Transition)		 시스템의 상태 변화와 이벤트 기반 테스트 로그인→메인→로그아웃
유스케이스
(Use Case)		 유스케이스를 기반으로 시나리오 테스트 사용자 행동 흐름

화이트박스 테스트 기법 (커버리지)

기법설명강도
구문 커버리지
(Statement Coverage)		 모든 구문(문장)이 1번 이상 실행 가장 약함
결정(분기) 커버리지
(Decision/Branch)		 모든 결정(분기)의 참/거짓이 1번 이상 실행 중간
조건 커버리지
(Condition Coverage)		 각 조건의 참/거짓이 1번 이상 실행 중간
조건/결정 커버리지
(MC/DC)		 조건 커버리지 + 결정 커버리지 동시 만족 강함
다중 조건 커버리지
(Multiple Condition)		 모든 조건의 가능한 참/거짓 조합 테스트 매우 강함
경로 커버리지
(Path Coverage)		 모든 실행 경로 테스트 가장 강함
커버리지 강도 순: 구문 < 결정 = 조건 < 조건/결정 < 다중조건 < 경로

통합 테스트 방식 비교 (자주 출제!)

방식특징필요 모듈장단점
빅뱅 통합 모든 모듈을 한꺼번에 통합 없음 + 단순
- 오류 발견 어려움
상향식 통합
(Bottom-Up)		 하위 모듈부터 통합 (잎 → 뿌리) 드라이버 필요 + 하위 모듈 먼저 검증
- 상위 설계 결함 늦게 발견
하향식 통합
(Top-Down)		 상위 모듈부터 통합 (뿌리 → 잎) 스텁 필요 + 상위 설계 조기 검증
- 하위 모듈 늦게 검증
샌드위치 통합 상향식 + 하향식 동시 적용 드라이버 + 스텁 + 병행 진행
- 복잡, 비용 높음
드라이버 (Driver): 상향식에서 아직 개발 안 된 상위 모듈을 대체하는 가상 모듈
스텁 (Stub): 하향식에서 아직 개발 안 된 하위 모듈을 대체하는 가상 모듈

결함(Defect) 용어 정리

용어정의
에러 (Error)사람의 실수 (개발자가 잘못 코딩)
결함 (Defect/Fault)에러로 인해 코드에 삽입된 오류
버그 (Bug)결함과 동의어로 사용
실패 (Failure)결함이 실행되어 발생하는 예상치 못한 동작

결함 심각도 분류

레벨설명
치명 (Critical)시스템 전체 중단, 데이터 손실
주요 (Major)핵심 기능 동작 불가
보통 (Minor)기능 일부 제한적 동작
경미 (Trivial)UI 불편, 미관 문제
