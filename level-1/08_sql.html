2 핵심 개념

SQL 4대 분류 (자주 출제!)

분류영문명령어특징
DDL Data Definition Language CREATE ALTER DROP TRUNCATE 구조 정의, 자동 COMMIT, ROLLBACK 불가
DML Data Manipulation Language SELECT INSERT UPDATE DELETE 데이터 조작, COMMIT/ROLLBACK 가능
DCL Data Control Language GRANT REVOKE 권한 부여·회수
TCL Transaction Control Language COMMIT ROLLBACK SAVEPOINT 트랜잭션 관리
DDL은 자동 COMMIT → ROLLBACK으로 되돌릴 수 없음!
TRUNCATE는 DDL, DELETE는 DML이므로 TRUNCATE는 롤백 불가

DDL - 주요 제약조건

제약조건설명
PRIMARY KEY기본키, NOT NULL + UNIQUE 자동 적용
FOREIGN KEY외래키, 참조 무결성 보장
UNIQUE중복 불허, NULL은 허용
NOT NULLNULL 값 불허
CHECK특정 조건 만족 여부 검사
DEFAULT값 미입력 시 기본값 설정

GRANT / REVOKE 문법 (필수 암기!)

WITH GRANT OPTION: 받은 권한을 다른 사용자에게도 부여 가능
CASCADE: 연쇄 권한 회수 (WITH GRANT OPTION으로 부여한 권한도 회수)

SELECT 절 구조 및 실행 순서

실행 순서: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
WHERE는 그룹화 전 필터 / HAVING은 그룹화 후 필터

서브쿼리 연산자

연산자반환 행설명
= 값단일행단일값 비교
IN (서브쿼리)다중행목록 내 포함 여부
ANY (서브쿼리)다중행하나라도 만족하면 TRUE
ALL (서브쿼리)다중행모두 만족해야 TRUE
EXISTS (서브쿼리)다중행결과가 하나라도 있으면 TRUE

집계 함수

함수설명NULL 처리
COUNT(*)전체 행 수NULL 포함
COUNT(컬럼)컬럼 행 수NULL 제외
SUM(컬럼)합계NULL 제외
AVG(컬럼)평균NULL 제외 후 계산
MAX/MIN최대/최소NULL 제외

JOIN 종류 요약

JOIN 종류결과키워드
INNER JOIN양쪽 모두 일치하는 행만JOIN 또는 INNER JOIN
LEFT OUTER JOIN왼쪽 전부 + 오른쪽 일치 (없으면 NULL)LEFT JOIN
RIGHT OUTER JOIN오른쪽 전부 + 왼쪽 일치 (없으면 NULL)RIGHT JOIN
FULL OUTER JOIN양쪽 전부 (없으면 NULL)FULL OUTER JOIN
CROSS JOIN카티션 곱 (모든 조합)CROSS JOIN
SELF JOIN같은 테이블을 두 번 사용별칭(AS) 활용

프로시저 / 함수 / 트리거 비교 (자주 출제!)

구분프로시저사용자 정의 함수트리거
생성CREATE PROCEDURECREATE FUNCTIONCREATE TRIGGER

RANK vs DENSE_RANK vs ROW_NUMBER (자주 출제!)

함수동점 처리예시 (90,90,80)
RANK() 동점 같은 순위, 다음 순위 건너뜀 1, 1, 3
DENSE_RANK() 동점 같은 순위, 다음 순위 연속 1, 1, 2
ROW_NUMBER() 무조건 고유 번호 1, 2, 3

그룹 함수 (ROLLUP / CUBE)

함수설명소계 생성
ROLLUP(A, B) 계층적 소계 생성 (A+B → A → 전체) 오른쪽→왼쪽 순서로 누적
CUBE(A, B) 모든 조합의 소계 생성 (A+B, A, B, 전체) 가능한 모든 조합
GROUPING SETS 원하는 그룹 기준을 직접 지정 지정한 항목만
