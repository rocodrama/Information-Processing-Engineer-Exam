2 핵심 개념

데이터 모델링 3단계

단계명칭표현 방법특징
1단계 개념적 모델링 ERD (Entity-Relationship Diagram) 현실 세계를 추상화, DBMS 독립적
2단계 논리적 모델링 관계형 스키마 (테이블, 속성, 키) DBMS 종류에 맞게 변환, 정규화 수행
3단계 물리적 모델링 실제 DB 스키마 (DDL) 인덱스, 파티션, 저장 구조 결정

ERD 구성요소 및 카디널리티

요소표기설명
개체 (Entity)□ 사각형데이터로 저장할 대상 (학생, 과목)
속성 (Attribute)○ 타원개체가 갖는 특성 (학번, 이름)
관계 (Relationship)◇ 마름모개체 간 연관성 (수강)
카디널리티의미예시M:N 처리
1 : 1하나 대 하나직원 : 사원증-
1 : N하나 대 여럿부서 : 직원-
M : N여럿 대 여럿학생 : 과목중간 테이블 생성 필수
M:N 관계는 관계형 DB에서 직접 표현 불가 → 중간(교차) 테이블로 1:N 두 개로 분리

이상현상 (Anomaly) — 정규화의 필요성

이상현상설명예시
삽입 이상
Insertion Anomaly		 데이터 삽입 시 불필요한 데이터도 함께 삽입해야 하는 현상 과목 없는 학생 등록 불가
삭제 이상
Deletion Anomaly		 데이터 삭제 시 의도치 않은 데이터도 함께 삭제되는 현상 학생 삭제 시 과목 정보도 삭제
갱신 이상
Update Anomaly		 데이터 수정 시 일부만 수정되어 데이터 불일치 발생 교수 이름 수정 시 일부 행만 변경

정규화 단계별 조건 (자주 출제!)

정규형조건 (= 제거 대상)
비정규 → 1NF 모든 속성값이 원자값(Atomic)이어야 함
다중값·반복 그룹 제거 (ex: 전화번호를 여러 개 가진 경우)
1NF → 2NF 부분 함수 종속 제거 (완전 함수 종속만 허용)
복합키 일부에만 종속된 속성을 분리
2NF → 3NF 이행 함수 종속 제거
A→B→C (A와 C의 이행 종속) 분리
3NF → BCNF 모든 결정자가 후보키여야 함
후보키가 아닌 속성이 결정자인 경우 분리
BCNF → 4NF 다치 종속 제거
4NF → 5NF 조인 종속 제거 (완전 정규화)
시험 범위: 주로 1NF → 2NF → 3NF → BCNF까지 출제
함수 종속 기호: A → B (A가 B를 결정: A를 알면 B를 알 수 있음)

함수 종속 예시

수강 테이블: (학번, 과목코드) → 성적, 과목명, 교수명, 학생명

종속 유형예시문제
완전 함수 종속 (학번, 과목코드) → 성적 정상 (2NF 만족)
부분 함수 종속 ← 2NF 위반 학번 → 학생명
과목코드 → 과목명		 복합키 일부에만 종속 → 분리
이행 함수 종속 ← 3NF 위반 과목코드 → 교수명
교수명 → 학과		 과목코드 → 학과 이행 종속 → 분리

반정규화 (Denormalization)

구분방법설명
테이블 테이블 병합 1:1, 1:N 관계 테이블을 합쳐 JOIN 제거
테이블 분할 자주 접근하는 컬럼과 그렇지 않은 컬럼 분리 (수직 분할)
컬럼 중복 컬럼 추가 자주 JOIN하는 테이블의 컬럼을 중복 저장
파생 컬럼 추가 계산값을 미리 저장 (합계, 평균 등)
반정규화는 조회 성능을 향상시키지만 데이터 무결성이 저하되고 저장 공간이 증가합니다.

인덱스 (Index)

구분설명
클러스터형테이블 데이터를 인덱스 순서로 물리적 정렬 (테이블당 1개)
비클러스터형별도 인덱스 구조 유지, 포인터로 데이터 참조 (여러 개 가능)
B-Tree 인덱스범위 검색에 효율적, 가장 일반적
Hash 인덱스등치(=) 검색에 효율적, 범위 검색 불가
인덱스 장점: SELECT 성능 향상
인덱스 단점: INSERT/UPDATE/DELETE 성능 저하, 추가 저장 공간 필요
인덱스 생성 기준: WHERE·JOIN·ORDER BY에 자주 사용되는 컬럼

뷰 (View) · 파티셔닝 (Partitioning)

구분설명
뷰 (View) 실제 데이터를 갖지 않는 가상 테이블
장점: 보안(일부 컬럼만 노출), 복잡한 쿼리 단순화
단점: 독립적인 인덱스 생성 불가, 삽입·수정·삭제 제한
수평 파티셔닝 행(Row) 기준 분할 (Range, Hash, List)
수직 파티셔닝 컬럼(Column) 기준 분할

트랜잭션 ACID 특성 (자주 출제!)

특성영문설명
원자성 Atomicity 트랜잭션은 전부 수행되거나 전혀 수행되지 않음 (All or Nothing)
COMMIT 또는 ROLLBACK
일관성 Consistency 트랜잭션 수행 후에도 DB가 일관된 상태를 유지
제약조건, 무결성 규칙 항상 만족
격리성 Isolation 동시 실행 중인 트랜잭션이 서로 간섭하지 않음
격리 수준: READ UNCOMMITTED ~ SERIALIZABLE
지속성 Durability 성공한 트랜잭션 결과는 영구적으로 저장
장애 후에도 COMMIT된 데이터는 유지

잠금(Lock)과 병행 제어

개념설명
공유 잠금 (S-Lock)읽기 전용 잠금, 여러 트랜잭션 동시 읽기 가능
배타 잠금 (X-Lock)읽기·쓰기 잠금, 다른 트랜잭션 접근 차단
교착상태 (Deadlock)두 트랜잭션이 서로의 잠금을 기다리는 상태
2단계 잠금 (2PL)직렬성 보장 프로토콜: 확장 단계 → 축소 단계

병행 제어 문제

문제설명
갱신 손실같은 데이터를 두 트랜잭션이 갱신할 때 하나가 손실
오손 읽기 (Dirty Read)ROLLBACK될 데이터를 읽음
반복 불가 읽기같은 쿼리를 두 번 실행하면 결과가 다름
유령 읽기 (Phantom Read)없던 행이 나타나거나 있던 행이 사라짐
