2 핵심 개념

모듈화 핵심 원칙

응집도 (Cohesion)

모듈 내부 요소들이 서로 얼마나 밀접하게 관련되어 있는가

높을수록 좋음 ↑
기능적 > 순차적 > ... > 우연적

결합도 (Coupling)

모듈 사이의 상호 의존 정도

낮을수록 좋음 ↓
내용 > 공통 > ... > 자료

응집도 7종류 (높음 → 낮음) 암기: 기순교절시논우

순위종류설명
1 (최고)기능적 응집도
Functional		모든 요소가 단 하나의 기능을 수행 (이상적)
2순차적 응집도
Sequential		한 요소의 출력이 다음 요소의 입력으로 사용
3교환적 응집도
Communicational		같은 데이터를 사용하는 요소들의 모임
4절차적 응집도
Procedural		정해진 순서대로 실행되는 요소들의 모임
5시간적 응집도
Temporal		특정 시간에 함께 실행되는 요소들 (초기화)
6논리적 응집도
Logical		유사한 성격의 요소들을 묶어놓음 (매개변수로 선택)
7 (최저)우연적 응집도
Coincidental		아무런 관계 없이 묶인 경우 (최악)

결합도 6종류 (높음 → 낮음) 암기: 내공외제스자

순위종류설명
1 (최고/최악)내용 결합도
Content		다른 모듈의 내부 데이터/코드를 직접 참조·수정
2공통 결합도
Common		전역 변수를 공유하는 경우
3외부 결합도
External		외부 환경(포트, 파일 형식 등)을 공유
4제어 결합도
Control		제어 신호(플래그)를 파라미터로 전달
5스탬프 결합도
Stamp		자료 구조(레코드/객체) 전체를 전달, 일부만 사용
6 (최저/최선)자료 결합도
Data		필요한 데이터만 매개변수로 전달 (이상적)

소프트웨어 아키텍처 패턴

패턴설명사용 예
계층화 패턴
Layered		 상위 계층이 하위 계층 서비스를 호출
각 계층은 인접 계층과만 통신		 OSI 7계층, MVC
파이프-필터
Pipe-Filter		 데이터를 필터(처리)가 순차 변환
파이프로 데이터 전달		 컴파일러, Unix 파이프
클라이언트-서버
Client-Server		 클라이언트가 요청, 서버가 응답
서버는 여러 클라이언트에 서비스		 웹, DB 서버
MVC 패턴 Model(데이터) + View(화면) + Controller(제어)
관심사 분리		 Spring MVC, Django
이벤트-버스
Event-Bus		 이벤트 소스→이벤트 버스→리스너
느슨한 결합		 Android 이벤트, Kafka
마스터-슬레이브
Master-Slave		 마스터가 슬레이브를 제어
병렬 처리 후 마스터가 합산		 DB 복제, 분산 계산

MVC 패턴 상세

구성요소역할예시
Model 데이터와 비즈니스 로직 관리
View/Controller와 독립적		 DB 엔티티, 서비스 로직
View 사용자 인터페이스 화면 담당
Model의 데이터를 시각화		 HTML 템플릿, JSP
Controller 사용자 요청을 받아 Model 업데이트
적절한 View 선택		 Servlet, @Controller
사용자 → Controller → Model → View → 사용자
장점: 관심사 분리, 유지보수성↑, 재사용성↑

형상관리 (Configuration Management) 4단계

단계활동설명
형상 식별CI 선정관리할 형상 항목(CI)을 식별하고 기준선 설정
형상 제어변경 관리변경 요청 검토 → 승인/거부 → 변경 수행
형상 상태 보고현황 기록형상 변경 이력, 현재 상태 문서화
형상 감사검증기준선과 현재 상태 비교 검증

버전 관리 도구 비교

방식분산형 (로컬 저장소)중앙집중형 (서버 저장소)

배치 프로그램 (Batch Program)

사용자 상호작용 없이 대량의 데이터를 자동으로 처리하는 프로그램

유형설명예시
이벤트 배치특정 이벤트 발생 시 자동 실행파일 도착 시 처리
온디맨드 배치사용자 요청에 따라 실행보고서 수동 생성
정기 배치설정된 시간에 정기적으로 실행매일 오전 2시 정산

배치 프로그램 구성 요소 (Spring Batch 기준)

구성요소설명

Job RepositoryJob 실행 이력 관리
Job LauncherJob을 실행시키는 컴포넌트
