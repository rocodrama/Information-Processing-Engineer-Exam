2 핵심 개념

UI 표준 vs UI 지침

구분설명
UI 표준전사적으로 공통 적용되는 화면 디자인/구성 규칙 (상위 개념)
UI 지침UI 표준을 구체적으로 적용하기 위한 세부 방법 (하위 개념)

UI 요구사항

사용자 인터뷰, 설문, 업무 분석 등을 통해 화면에 필요한 기능적·비기능적 요구사항을 수집하고 정리합니다.

프로토타입 · 와이어프레임 · 스토리보드 비교 (자주 출제!)

구분설명상세도
와이어프레임화면의 레이아웃을 간단한 선/도형으로 스케치, 디자인 요소는 배제낮음
스토리보드화면 단위로 흐름·상호작용·처리 로직까지 포함한 상세 설계 문서높음 (개발 전달용)
프로토타입실제 클릭/동작이 가능한 형태로 제작하여 사용자 검증에 활용실제 화면과 유사

화면 흐름 / 폼 흐름

구분설명
화면 흐름도화면 간 이동 순서와 전환 조건을 표현한 다이어그램
폼 흐름입력 폼 간 이동 절차와 데이터 처리 순서를 정의

메뉴 구조

기능을 계층적(트리형)으로 분류하여 사용자가 원하는 기능을 빠르게 찾을 수 있도록 설계합니다.

사용자 편의성

요소설명
접근성다양한 사용자(장애인 포함)가 쉽게 이용할 수 있도록 설계
가시성중요한 정보와 기능이 잘 보이도록 배치
피드백 제공사용자 행동에 대해 시스템이 즉시 결과를 알려줌

UI 설계 원칙 (자주 출제!)

원칙설명
직관성사용자가 학습 없이도 쉽게 이해하고 사용할 수 있어야 함
일관성동일한 기능은 동일한 디자인/위치/동작으로 제공해야 함
유효성(효율성)정확하고 완전하게 사용자의 목표를 달성할 수 있도록 함
유연성다양한 사용자 요구와 환경 변화에 대응할 수 있어야 함
학습성누구나 쉽게 배우고 익힐 수 있어야 함
