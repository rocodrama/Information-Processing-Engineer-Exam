03 IP 계산 공식 & 단계별 풀이

핵심 공식 4가지

호스트 수 (사용 가능) = 2^(32−prefix) − 2

블록 크기 = 2^(호스트 비트 수) = 256 − 마지막 옥텟 마스크 값

네트워크 주소 = IP AND 서브넷 마스크

브로드캐스트 = 네트워크 주소 + (블록 크기 − 1)
📌 예제: IP 223.13.234.132 / 마스크 255.255.255.192
1
Prefix 구하기

192 = 11000000₂ → 켜진 비트 수: 24 + 2 = /26

2
호스트 비트 수

32 − 26 = 6비트

3
블록 크기

2⁶ = 64  (또는 256 − 192 = 64)

4
네트워크 주소

132 ÷ 64 = 2.xxx → 내림 → 64 × 2 = 128 → 네트워크: 223.13.234.128

5
브로드캐스트

128 + 64 − 1 = 191 → 브로드캐스트: 223.13.234.191

6
사용 가능한 호스트 수

2⁶ − 2 = 62개  (네트워크 주소, 브로드캐스트 제외)

결과 정리
네트워크: 223.13.234.128  |  브로드캐스트: 223.13.234.191  |  호스트: 223.13.234.129 ~ .190 (62개)
💡 블록 크기로 네트워크 주소 찾기: IP의 마지막 옥텟 ÷ 블록 크기 = 몫 → 몫 × 블록 크기 = 네트워크 주소의 마지막 옥텟
예) 132 ÷ 64 = 2.0625 → 몫 2 → 2 × 64 = 128
04 FLSM 서브넷 분할
FLSM(Fixed Length Subnet Mask): 동일한 크기의 서브넷으로 분할. 몇 개의 서브넷으로 나눌지 주어지면 필요한 비트 수를 계산합니다.
📌 예제: 192.168.1.0/24 → 3개 서브넷으로 분할
1
필요한 서브넷 비트 수

2^n ≥ 3 → 2²=4 ≥ 3 → 2비트 필요

2
새 Prefix

/24 + 2 = /26 → 마스크: 255.255.255.192

3
블록 크기

2^(32−26) = 2⁶ = 64

4
각 서브넷 범위

서브넷 1: 192.168.1.0 ~ 192.168.1.63  (브로드캐스트: .63)
서브넷 2: 192.168.1.64 ~ 192.168.1.127  (브로드캐스트: .127 ← 기출 정답)
서브넷 3: 192.168.1.128 ~ 192.168.1.191  (브로드캐스트: .191)
서브넷 4: 192.168.1.192 ~ 192.168.1.255  (미사용)

FLSM 공식 정리

서브넷 비트 수 n = ⌈log₂(필요 서브넷 수)⌉

새 Prefix = 기존 Prefix + n

k번째 서브넷 시작 주소 마지막 옥텟 = (k−1) × 블록 크기

k번째 서브넷 브로드캐스트 = k × 블록 크기 − 1

08 네트워크 공격 유형
SYN Flooding
TCP 3-way handshake 취약점 악용. SYN 패킷을 대량 전송 후 ACK를 보내지 않아 서버의 연결 대기 큐를 가득 채움 → DoS
스머프 (Smurf)
ICMP Echo Request를 브로드캐스트 주소로 전송, 출발지 IP를 피해자 IP로 위조 → 수많은 호스트가 피해자에게 Echo Reply 전송
ARP Spoofing
ARP 응답을 위조하여 자신의 MAC 주소를 게이트웨이로 등록 → 중간자(MitM) 공격, 트래픽 도청
스니핑 (Sniffing)
네트워크 패킷을 수동적으로 도청·수집. 암호화되지 않은 데이터에서 ID/PW 탈취. 수동적 공격
스푸핑 (Spoofing)
IP·MAC·이메일 등을 위조하여 신뢰할 수 있는 호스트인 척 공격. 능동적 공격
세션 하이재킹
TCP 시퀀스 번호를 예측하여 이미 인증된 세션 탈취. 재인증 없이 권한 획득
XSS
웹 페이지에 악성 스크립트 삽입 → 다른 사용자의 브라우저에서 실행됨. 쿠키 탈취·세션 도용
SQL Injection
입력값에 SQL 구문을 삽입하여 DB 조작. 인증 우회·데이터 유출·삭제 가능
CSRF
사용자가 의도하지 않은 요청을 강제로 전송. 이미 로그인된 사용자의 권한 악용
DoS / DDoS
서비스 거부 공격. DoS: 단일 출처. DDoS: 다수 좀비PC(봇넷) 이용 분산 공격
