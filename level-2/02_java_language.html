상속 + super 키워드 추적

 1class Parent {
 2    int x = 10;
 3    void show() { System.out.print("P:" + x + " "); }
 4}
 5class Child extends Parent {
 6    int x = 20;
 7    @Override
 8    void show() {
 9        super.show();           // Parent의 show()
10        System.out.print("C:" + x + " ");     // Child의 x
11        System.out.print("S:" + super.x);    // Parent의 x
12    }
13}
14new Child().show();
실행 순서호출 대상출력
①Child.show() 진입
②super.show() → Parent의 show(): x=10 (Parent의 x)P:10
③Child.x = 20 출력C:20
④super.x = Parent.x = 10 출력S:10
✅ super.show() 내부에서 x는 Parent의 x(10). Child의 x(20)는 Child 스코프에서만 보임.
02 다형성 — 메서드 vs 필드 바인딩
🚨 시험 최빈출 함정! 업캐스팅 시 메서드는 자식 클래스 것이 실행되지만, 필드는 선언 타입(부모) 기준으로 접근된다.
구분
결과
메서드 호출 obj.method()
→ 실제 타입(Child) 메서드 실행
런타임 동적 바인딩
필드 접근 obj.field
→ 선언 타입(Parent) 필드 접근
컴파일 타임 정적 바인딩

업캐스팅 다형성 추적

 1class A {
 2    String s = "A-field";
 3    String name() { return "A"; }
 4}
 5class B extends A {
 6    String s = "B-field";
 7    @Override
 8    String name() { return "B"; }
 9}
10A obj = new B();  // 업캐스팅: 선언타입=A, 실제타입=B
11System.out.println(obj.name()); // 메서드: B의 name()
12System.out.println(obj.s);      // 필드: A의 s
라인접근 대상바인딩출력
11obj.name()동적 → B의 name()B
12obj.s정적 → A의 sA-field
03 생성자 체이닝
this(): 같은 클래스의 다른 생성자 호출 (반드시 첫 줄)
super(): 부모 클래스 생성자 호출 (명시 없으면 자동으로 super() 삽입)
실행 순서: 부모 생성자 → 자식 생성자 (위에서 아래로)

생성자 체이닝 추적

 1class A {
 2    A() {
 3        this(10);                      // → A(10) 호출
 4        System.out.print("A0 ");
 5    }
 6    A(int n) {
 7        System.out.print("A" + n + " "); // A10
 8    }
 9}
10class B extends A {
11    B() {
12        super();                        // → A() 호출
13        System.out.print("B ");
14    }
15}
16new B();
호출 순서 (들어가기)
new B()→ B() 진입
B(): super()→ A() 진입
A(): this(10)→ A(10) 진입
A(10)→ 출력: "A10 " 후 리턴
리턴 순서 (나오기)
A(): this(10) 후→ 출력: "A0 " 후 리턴
B(): super() 후→ 출력: "B " 후 리턴
✅ 최종 출력: A10 A0 B
04 예외처리 — try-catch-finally
⚠️ finally는 반드시 실행됨! return이 있어도, 예외가 발생해도, catch가 처리해도 finally는 무조건 실행.
상황trycatchfinally출력 순서
예외 없음전체 실행실행 안됨실행됨try → finally
예외 발생, catch 처리예외 지점까지만실행됨실행됨try(부분) → catch → finally
예외 발생, catch 없음예외 지점까지만—실행됨try(부분) → finally → 예외전파

예외처리 실행 순서 추적

 1try {
 2    System.out.print("A");        // ① 실행
 3    int r = 10 / 0;              // ② ArithmeticException 발생!
 4    System.out.print("B");        // ③ 실행 안됨 (건너뜀)
 5} catch (ArithmeticException e) {
 6    System.out.print("C");        // ④ 실행 (예외 처리)
 7} finally {
 8    System.out.print("D");        // ⑤ 무조건 실행
 9}
A (try 시작)
→
10/0 예외!
→
B (건너뜀)
→
C (catch)
→
D (finally)
✅ 출력: ACD  (B는 예외 발생 후 건너뜀)

try 안에 return이 있어도 finally 실행

 1static int test() {
 2    try {
 3        System.out.print("T");
 4        return 1;          // return 전에 finally 먼저!
 5    } finally {
 6        System.out.print("F"); // return보다 먼저 실행
 7    }
 8}
 9test(); // 출력: TF, 반환값: 1
05 static 변수 & 메서드
static 멤버는 클래스 단위로 하나만 존재. 모든 인스턴스가 공유.
인스턴스 없이 ClassName.member로 직접 접근 가능.

static 변수 공유 추적

 1class Counter {
 2    static int count = 0;   // 클래스 공유 변수
 3    int id;
 4    Counter() {
 5        count++;
 6        id = count;
 7    }
 8}
 9Counter a = new Counter();
10Counter b = new Counter();
11Counter c = new Counter();
12System.out.println(Counter.count);  // ?
13System.out.println(a.id + b.id + c.id); // ?
이벤트count (공유)인스턴스.id
new Counter() for a1a.id = 1
new Counter() for b2b.id = 2
new Counter() for c3c.id = 3
Counter.count3
a.id + b.id + c.id1 + 2 + 3 = 6
06 인터페이스 & 추상클래스
구분추상클래스 (abstract)인터페이스 (interface)
상속단일 상속 (extends)다중 구현 (implements)
생성자있음 (직접 객체화 불가)없음
일반 메서드가능default 메서드로만 가능 (Java 8+)
필드일반 필드 가능public static final 상수만

인터페이스 default 메서드 추적

 1interface Greet {
 2    default String hello() { return "Hi"; }
 3    String name();              // 추상 메서드 (구현 필수)
 4}
 5class Korean implements Greet {
 6    @Override
 7    public String name() { return "Korea"; }
 8    // hello()는 default 사용 → "Hi"
 9}
10Greet g = new Korean();
11System.out.println(g.hello() + g.name());
✅ 출력: HiKorea
Korean이 hello()를 오버라이드하지 않았으므로 Greet의 default hello()가 실행됨.
07 람다식 (Lambda)
람다식 = 함수형 인터페이스(추상 메서드가 1개인 인터페이스)의 구현체를 간결하게 표현.
(매개변수) -> 실행문 또는 (매개변수) -> { 실행블록; return 값; }
함수형 인터페이스추상 메서드람다 예시
Runnablevoid run()() -> System.out.print("Hi")
Supplier<T>T get()() -> 42
Consumer<T>void accept(T)x -> System.out.print(x)
Function<T,R>R apply(T)x -> x * 2
Predicate<T>boolean test(T)x -> x > 0

사용자 정의 함수형 인터페이스 + 람다 추적

 1interface Calc {
 2    int operate(int a, int b);  // 추상 메서드 1개
 3}
 4Calc add = (a, b) -> a + b;
 5Calc mul = (a, b) -> a * b;
 6Calc max = (a, b) -> (a > b) ? a : b;

08 기출 문제
기출 빈출  다음 Java 코드의 출력 결과를 쓰시오. (상속 + 오버라이딩 + super)
▼ 
class Animal {
    String type = "동물";
    void sound() { System.out.print("..."); }
}
class Dog extends Animal {
    String type = "강아지";
    @Override
    void sound() {
        System.out.print(super.type + ":");
        System.out.print("멍");
    }
}
public class Main {
    public static void main(String[] args) {
        Animal a = new Dog();
        a.sound();
        System.out.print(" " + a.type);
    }
}
정답
동물:멍 동물
a.sound() → 동적 바인딩 → Dog의 sound() 실행
super.type = Animal의 type = "동물", 이어서 "멍" 출력 → 동물:멍
a.type → 정적 바인딩 → Animal의 type = "동물"
기출 빈출  다음 Java 코드의 출력 결과를 쓰시오. (try-catch-finally)
▼ 
public class Main {
    public static void main(String[] args) {
        try {
            System.out.print("A");
            if (1 == 1) throw new Exception();
            System.out.print("B");
        } catch (Exception e) {
            System.out.print("C");
            return;
        } finally {
            System.out.print("D");
        }
        System.out.print("E");
    }
}
정답
ACD
A 출력 → throw Exception → B 건너뜀 → catch → C 출력 → return 실행 직전 finally → D 출력 → 메서드 종료 → E 실행 안됨
catch의 return이 있어도 finally는 반드시 실행됨.

class Box {

15
7

Box.total = 15, b.size = 7
2024 1회 Q4 유사  다음 Java 코드의 출력 결과를 쓰시오. (문자열 인덱스 역순)
▼ 
public class Main {
    public static void main(String[] args) {
        String str = "ABCDEFG";
        String result = "";
        for (int i = str.length() - 1; i >= 0; i -= 2) {
            result += str.charAt(i);
        }
        System.out.print(result);
    }
}
정답
GECA
str = "ABCDEFG" (인덱스 0~6)
i=6: charAt(6)='G' → i=4: charAt(4)='E' → i=2: charAt(2)='C' → i=0: charAt(0)='A' → i=-2: 종료
result = "G"+"E"+"C"+"A" = GECA
개념 확인  다음 Java 코드의 출력 결과를 쓰시오. (예외 catch 순서 — 좁은 예외 먼저)
▼ 
public class Main {
    public static void main(String[] args) {
        try {
            int[] arr = new int[3];
            arr[5] = 10;  // ArrayIndexOutOfBoundsException
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.print("범위초과");
        } catch (RuntimeException e) {
            System.out.print("런타임");
        } catch (Exception e) {
            System.out.print("예외");
        } finally {
            System.out.print("+종료");
        }
    }
}
정답
범위초과+종료
ArrayIndexOutOfBoundsException 발생 → 첫 번째 catch가 정확히 일치 → "범위초과" 출력
catch는 위에서 아래로 순서대로 매칭, 먼저 걸리면 나머지는 실행 안됨.
⚠️ Exception(부모)을 먼저 쓰면 컴파일 에러! → 좁은 예외 타입을 항상 먼저 작성.
기출 유사  다음 Java 코드의 출력 결과를 쓰시오. (추상 클래스 + 오버라이딩)
▼ 
abstract class Shape {
    abstract double area();
    void print() {
        System.out.printf("넓이=%.0f\n", area());
    }
}
class Rectangle extends Shape {
    int w, h;
    Rectangle(int w, int h) { this.w=w; this.h=h; }
    @Override
    double area() { return w * h; }
}
public class Main {
    public static void main(String[] args) {
        Shape s = new Rectangle(5, 4);
        s.print();
    }
}
정답
넓이=20
s.print() → Shape의 print() 실행 → 내부에서 area() 호출
area()는 동적 바인딩 → Rectangle의 area() = 5×4 = 20
printf("넓이=%.0f\n", 20.0) → 넓이=20
기출 유사  다음 Java 람다식 코드의 출력 결과를 쓰시오.
▼ 
import java.util.function.*;

public class Main {
    public static void main(String[] args) {
        Function<Integer, Integer> dbl = x -> x * 2;
        Function<Integer, Integer> sq  = x -> x * x;

        int a = dbl.apply(3);           // 3*2 = 6
        int b = sq.apply(a);            // 6*6 = 36
        int c = dbl.andThen(sq).apply(3); // dbl(3)=6, sq(6)=36

        System.out.println(a + " " + b + " " + c);
    }
}
정답
6 36 36
dbl.apply(3) = 3×2 = 6 → a=6
sq.apply(6) = 6×6 = 36 → b=36
dbl.andThen(sq).apply(3): 먼저 dbl(3)=6, 그 결과를 sq(6)=36 → c=36
출력: 6 36 36
개념 확인  오버로딩(Overloading)과 오버라이딩(Overriding)의 차이를 설명하시오.
▼
정답
매개변수 다름 vs 상속 후 재정의
오버로딩 (Overloading): 같은 클래스에서 메서드 이름은 같고, 매개변수(타입/개수)가 다른 메서드를 여러 개 정의.
→ print(int), print(String), print(int, int)

오버라이딩 (Overriding): 상속 관계에서 부모 메서드를 자식이 재정의. 이름·매개변수·반환타입 모두 동일, 구현만 변경.
→ @Override void speak() { ... }
