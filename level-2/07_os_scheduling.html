01 스케줄링 핵심 공식

반드시 외울 공식 4가지

반환시간 (Turnaround) = 완료시간 − 도착시간

대기시간 (Waiting) = 반환시간 − 실행시간(Burst)

평균 대기시간 = 모든 프로세스 대기시간 합 ÷ 프로세스 수

응답시간 (Response) = 첫 번째 CPU 할당 시작 − 도착시간
💡 간트차트 먼저 그리기: 스케줄링 문제는 반드시 간트차트를 그린 뒤 각 프로세스의 완료시간을 구하세요. 완료시간만 알면 나머지는 공식으로 계산됩니다.

03 SJF & SRT
SJF
실행시간이 가장 짧은 프로세스 먼저 실행.
비선점 — 실행 중인 프로세스는 끝날 때까지 교체 안 함.
비선점
SRT
남은 실행시간이 가장 짧은 프로세스 우선.
새 프로세스 도착 시 남은 시간 비교 후 선점 가능.
선점 (SJF의 선점 버전)

05 스케줄링 알고리즘 비교
알고리즘선점기준특징단점
FCFS비선점도착 순서구현 단순Convoy Effect (긴 작업이 짧은 작업 막음)
SJF비선점실행시간 짧은 순평균 대기시간 최소기아(Starvation) 발생 가능, 실행시간 예측 어려움
SRT선점남은 시간 짧은 순SJF의 선점 버전문맥교환 오버헤드, 기아 발생
우선순위선점/비선점우선순위 높은 순중요 작업 빠른 처리기아 → 에이징(Aging)으로 해결
RR선점도착 순서 + 타임퀀텀공평성, 응답시간 우수타임퀀텀에 따라 성능 차이, 문맥교환 오버헤드
다단계 큐선점큐 우선순위여러 큐 계층 구조큐 간 이동 없어 기아 가능
06 페이지 교체 알고리즘 개요
메모리 프레임이 꽉 찬 상태에서 새 페이지를 적재할 때 어떤 페이지를 내보낼지 결정하는 알고리즘. 페이지 부재(Page Fault) 횟수가 적을수록 좋음.
FIFO
가장 먼저 들어온 페이지 교체.
구현 단순. Belady's Anomaly 발생 가능 (프레임 늘려도 부재 증가).
큐 구조
LRU
가장 오래 전에 사용된 페이지 교체.
최근 사용 이력 기반. 실제 OS에서 많이 사용.
최근 미사용
OPT
앞으로 가장 오래 사용되지 않을 페이지 교체.
이론적으로 최적. 미래 참조를 알아야 하므로 실제 구현 불가.
이론상 최적
LFU
사용 빈도가 가장 낮은 페이지 교체.
빈도 동일 시 FIFO 또는 LRU 적용.
최저 빈도

10 교착상태 (Deadlock)
암기: 상점비순 — 상호배제, 점유대기, 비선점, 순환대기 (4가지 조건 모두 성립할 때만 교착상태 발생)
조건 1
상호배제
자원은 한 번에 하나의 프로세스만 사용 가능
조건 2
점유대기
자원을 점유한 채 다른 자원을 기다림
조건 3
비선점
점유된 자원을 강제로 빼앗을 수 없음
조건 4
순환대기
프로세스들이 원형으로 서로의 자원을 기다림
처리 방법설명
예방 (Prevention)4가지 조건 중 하나를 제거. 자원 낭비 심함
회피 (Avoidance)안전 상태 유지 — 은행원 알고리즘(Banker's Algorithm)
탐지 (Detection)교착 발생 후 탐지 → 자원 할당 그래프
회복 (Recovery)프로세스 종료 또는 자원 선점하여 해제
