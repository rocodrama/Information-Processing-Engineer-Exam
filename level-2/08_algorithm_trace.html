08 스택 (Stack)

스택 — LIFO (Last In First Out)

PUSH 1[1] ← top
PUSH 2[1, 2] ← top
PUSH 3[1, 2, 3] ← top
POP반환=3 / [1, 2] ← top
PUSH 4[1, 2, 4] ← top
POP반환=4 / [1, 2] ← top
POP반환=2 / [1] ← top
POP반환=1 / [] (empty)
POP 순서: 3 → 4 → 2 → 1

스택 활용

▸ 함수 호출 스택 (Call Stack)
▸ 괄호 짝 검사
▸ 중위→후위 표기 변환
▸ 뒤로 가기 (브라우저)
▸ DFS (깊이 우선 탐색)
▸ Undo 기능
09 큐 & 원형 큐 (Queue)

큐 — FIFO (First In First Out)

ENQ 1front[1]rear
ENQ 2front[1,2]rear
ENQ 3front[1,2,3]rear
DEQ반환=1 / front[2,3]rear
ENQ 4front[2,3,4]rear
DEQ반환=2 / front[3,4]rear
DEQ 순서: 1 → 2 (삽입 순서대로)

큐 활용

▸ 프로세스 스케줄링 (RR)
▸ 프린터 스풀링
▸ BFS (너비 우선 탐색)
▸ 버퍼 (네트워크, IO)
▸ 메시지 큐
📌 원형 큐 (Circular Queue) — 2025 2회 Q12 기출
front와 rear가 배열 크기를 넘으면 처음으로 되돌아감. rear = (rear + 1) % size 로 순환. 빈 공간 구분을 위해 실제 크기보다 1칸 작게 사용.
1
큐 크기 = 5 (인덱스 0~4), 초기 front=rear=0
2
ENQ 1,2,3,4 → rear=4, front=0
배열: [_, 1, 2, 3, 4] (인덱스 0은 비워둠)
3
DEQ 2번 → front=(0+1)%5=1, front=(1+1)%5=2
1, 2가 반환되고 front=2
4
ENQ 5 → rear=(4+1)%5=0
배열: [5, _, 2는deq, 3, 4] → 실제: 3, 4, 5 남음
✅ 원형 큐 조건
빈 큐: front == rear
꽉 찬 큐: (rear + 1) % size == front

▼
정답
전위: A B D E C F G
중위: D B E A F C G
후위: D E B F G C A
전위(루트→왼→오): A → (B → D → E) → (C → F → G)
중위(왼→루트→오): (D → B → E) → A → (F → C → G)
후위(왼→오→루트): (D → E → B) → (F → G → C) → A
개념 확인  다음 정렬 알고리즘 중 안정 정렬(Stable Sort)에 해당하는 것을 모두 고르시오.
버블정렬 / 선택정렬 / 삽입정렬 / 퀵정렬 / 합병정렬 / 힙정렬
▼
정답
버블정렬, 삽입정렬, 합병정렬
안정 정렬: 동일 키 값의 상대 순서 유지. 버블·삽입·합병·기수 = 안정. 선택·퀵·힙 = 불안정.
개념 확인  다음 Big-O 표기를 빠른 순서대로 나열하시오.
O(n²), O(log n), O(n!), O(n log n), O(1), O(n), O(2ⁿ)
▼
정답
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)
상수 → 로그 → 선형 → 선형로그 → 이차 → 지수 → 팩토리얼 순으로 느려짐.
개념 확인  퀵정렬의 최선·평균·최악 시간복잡도를 쓰고, 최악이 발생하는 경우를 설명하시오.
▼
정답
최선: O(n log n) / 평균: O(n log n) / 최악: O(n²)
최악 발생 조건: 이미 정렬된 데이터에서 첫 번째(또는 마지막) 원소를 피벗으로 선택할 때. 매번 분할이 1:n-1로 치우쳐 n번 재귀 → O(n²). 랜덤 피벗 또는 중간값 피벗 선택으로 완화.
