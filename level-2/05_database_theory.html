02 함수적 종속 (Functional Dependency)
X → Y: X 값이 결정되면 Y 값이 유일하게 결정됨 (X가 Y를 함수적으로 결정)
기본키는 모든 비키 속성을 함수적으로 결정해야 함.

종속 유형 — 수강 테이블 예시

기본키: {학번, 과목코드}
{학번, 과목코드} → 성적 완전 함수 종속 — PK 전체에만 의존
학번 → 이름 부분 함수 종속 — PK 일부만으로 결정
과목코드 → 과목명, 교수번호 부분 함수 종속 — PK 일부만으로 결정

05 트랜잭션 & ACID
트랜잭션: 데이터베이스의 상태를 변화시키는 논리적인 작업 단위. 완전히 수행되거나(COMMIT), 전혀 수행되지 않거나(ROLLBACK).
A
Atomicity
원자성
트랜잭션 내 연산은 모두 성공하거나 모두 실패. 중간 상태 없음.
→ 은행 이체: 출금 성공 + 입금 실패 = 전체 취소
C
Consistency
일관성
트랜잭션 전후 데이터베이스가 일관된 상태 유지.
→ 제약조건, 무결성 규칙 항상 만족
I
Isolation
격리성(독립성)
동시 실행 트랜잭션은 서로 간섭 없이 독립적 실행.
→ 다른 트랜잭션 중간 결과를 참조 불가
D
Durability
영속성(지속성)
COMMIT된 트랜잭션 결과는 영구 저장.
→ 시스템 오류 후 재시작해도 보존

08 기출 문제
기출 빈출  다음 테이블에서 발생하는 이상현상의 이름을 쓰시오.
학생(학번PK, 이름, 과목코드PK, 과목명, 성적) — S3 행이 유일하게 C3 정보 보유 중
▼
정답
① 삭제 이상 (Deletion Anomaly)
S3 행을 삭제하면 해당 학생 정보와 함께 C3 과목 정보도 사라지는 삭제 이상.
→ 원인: 학생 정보와 과목 정보가 한 테이블에 혼재 (정규화 필요)

나머지 이상현상도 함께 암기:
삽입 이상: 특정 정보만 삽입 불가 (PK 없이 삽입 안됨)
갱신 이상: 같은 데이터 여러 곳 수정 → 일부만 변경 시 불일치
기출 빈출  다음 테이블이 2NF를 만족하지 않는 이유와 2NF로 분해하시오.
▼
주문(주문번호, 상품코드, 고객번호, 상품명, 고객명, 수량)
PK: {주문번호, 상품코드}
FD: 상품코드 → 상품명  |  고객번호 → 고객명  |  {주문번호, 상품코드} → 수량, 고객번호
정답
부분 함수 종속 존재 → 2NF 위반
위반 이유: 상품코드만으로 상품명 결정 가능 (PK 전체가 아닌 일부에 종속)

2NF 분해:
① 주문(주문번호, 상품코드, 고객번호, 수량) — 완전 종속 속성만
② 상품(상품코드, 상품명) — 상품코드에 부분 종속된 것 분리
③ 고객(고객번호, 고객명) — 고객번호에 부분 종속된 것 분리
기출 빈출  다음 FD가 주어질 때 이 테이블이 몇 정규형인지 쓰시오.
직원(사번, 부서코드, 부서명, 프로젝트, 업무) / PK: {사번, 프로젝트} / FD: 사번→부서코드, 부서코드→부서명, {사번,프로젝트}→업무
▼
정답
1NF (1정규형)
원자값을 가지므로 1NF는 만족.
그러나 부분 함수 종속 존재: 사번만으로 부서코드 결정 (PK 일부에 종속) → 2NF 위반
이행 함수 종속도 존재: 사번→부서코드→부서명 → 3NF도 위반
따라서 가장 높은 만족 정규형 = 1NF
기출 빈출  트랜잭션의 ACID 특성 중 "일관성(Consistency)"이 의미하는 바를 설명하시오.
▼
정답
트랜잭션 수행 전후 데이터베이스가 일관된(정확한) 상태 유지
트랜잭션 실행 전 데이터베이스가 무결성 제약조건을 만족한 상태라면, 트랜잭션 실행 후에도 반드시 무결성 제약조건을 만족해야 함.
예: 계좌 총액은 이체 전후 동일해야 함 (출금액 = 입금액)
기출 빈출  교착상태(Deadlock) 발생의 필요충분조건 4가지를 쓰시오.
▼
정답
상호배제, 점유와 대기, 비선점, 환형 대기
암기법: 상점비환
① 상호배제: 자원은 한 번에 하나의 프로세스만 사용
② 점유와 대기: 자원을 점유한 채 다른 자원 대기
③ 비선점: 다른 프로세스의 자원을 강제로 빼앗을 수 없음
④ 환형 대기: 프로세스가 원형 체인을 이루며 서로 대기
→ 4가지 조건 중 하나라도 제거하면 교착상태 예방 가능
기출 유사  다음 중 참조 무결성을 위반하는 경우를 모두 고르시오.
부서(부서코드 PK) / 직원(사번 PK, 부서코드 FK → 부서.부서코드)
▼
① 직원 테이블에 존재하지 않는 부서코드를 FK로 가진 행 삽입
② 직원 FK를 NULL로 설정
③ 직원이 참조 중인 부서 행을 CASCADE 없이 삭제
④ 부서 테이블에 새 행 삽입
정답
① ③
① 존재하지 않는 부서코드 참조 → 참조 무결성 위반
② FK = NULL은 허용됨 (참조 없음을 의미)
③ 직원이 참조 중인 부서 삭제 → RESTRICT면 거부 또는 에러 → 위반 시도
④ 부모 테이블(부서)에 새 행 추가는 문제 없음
개념 확인  3NF와 BCNF의 차이를 설명하고, BCNF이면 반드시 3NF인지 여부를 쓰시오.
▼
정답
BCNF ⊂ 3NF (BCNF이면 3NF이지만 역은 성립 안 함)
3NF: X→Y에서 Y가 비키 속성일 때, X가 슈퍼키이거나 Y가 후보키의 일부여야 함
BCNF: X→Y(Y⊄X)에서 X는 반드시 슈퍼키여야 함 (더 강한 조건)

→ BCNF를 만족하면 3NF도 만족 (BCNF가 더 강함)
→ 3NF를 만족해도 BCNF를 위반할 수 있음
→ 정규화 단계: 비정규형 < 1NF < 2NF < 3NF < BCNF < 4NF < 5NF

정답
Undo: COMMIT 안된 트랜잭션 취소
Redo: COMMIT 완료된 트랜잭션 재실행
장애 발생 후 복구 시:
Undo (되돌리기): 장애 시점에 아직 COMMIT되지 않은 트랜잭션의 변경사항 취소
→ 원자성(Atomicity) 보장

Redo (재실행): 이미 COMMIT 완료했지만 디스크에 반영 안된 트랜잭션 재실행
→ 영속성(Durability) 보장
