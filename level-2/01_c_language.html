02 배열 & 포인터 관계
배열 이름은 첫 번째 원소의 주소. arr[i] 와 *(arr+i) 는 완전히 동일. 2차원 배열의 경우 arr[i][j] = *(*(arr+i)+j).

1차원 배열 포인터 추적

1int arr[] = {10, 20, 30, 40, 50};
2int *p = arr;    // p = &arr[0]
3printf("%d\n", *p);       // 10
4printf("%d\n", *(p+2));   // 30
5printf("%d\n", p[3]);     // 40  (p[i] == *(p+i))
6p++;                       // p → arr[1]
7printf("%d\n", *p);       // 20
✅ 핵심: p++ 하면 p는 다음 int 주소로 이동 (4바이트 전진). *(p+n) = p[n] = arr[기준+n]

2차원 배열 포인터 추적 — 기출 빈출 패턴

1int arr[3][4] = {
2    {1,  2,  3,  4},   // index 0~3
3    {5,  6,  7,  8},   // index 4~7
4    {9, 10, 11, 12}    // index 8~11
5};
6int *p = arr[0];   // p = &arr[0][0] = 1의 주소
7printf("%d", *(p+6) + *(p+2));  // ?
표현식오프셋행/열값
*(p+0)0arr[0][0]1
*(p+2)2arr[0][2]3
*(p+4)4arr[1][0]5
*(p+6)6arr[1][2]7
*(p+6) + *(p+2)7 + 3 = 10
💡 오프셋 계산법: p = arr[0][0]부터 시작. 열 수(4)를 기준으로 행을 건너뜀.
arr[i][j] = *(p + i×4 + j)

04 재귀함수 (Recursion)
⚠️ 재귀 추적의 핵심: 호출 스택을 직접 쌓아보기. 각 호출이 반환하는 값을 바깥 호출에 대입하며 역으로 계산.

팩토리얼 재귀 — f(5) 추적

1int f(int n) {
2    if (n <= 1) return 1;   // 기저 조건
3    return n * f(n - 1);
4}
▼ 호출 스택 (내려가기)
f(5)→ 5 × f(4)
f(4)→ 4 × f(3)
f(3)→ 3 × f(2)
f(2)→ 2 × f(1)
f(1)→ return 1
▲ 반환 (올라오기)
f(5)= 5 × 24 = 120
f(4)= 4 × 6 = 24
f(3)= 3 × 2 = 6
f(2)= 2 × 1 = 2
f(1)= 1
✅ f(7) = 7! = 7×6×5×4×3×2×1 = 5040 — 2023 3회 Q8 기출 정답
05 static 변수
static 변수는 함수가 끝나도 값이 유지됨. 초기화는 프로그램 시작 시 단 한 번만 실행. 전역 변수처럼 메모리에 남아있지만 접근 범위는 함수 내부로 제한.

static 변수 누적 추적

1#include <stdio.h>
2int add() {
3    static int s = 0;  // 딱 한 번만 초기화
4    s += 5;
5    return s;
6}
7int main() {
8    add(); add(); add();
9    printf("%d", add());   // ?
10   return 0;
11}
호출 횟수s (이전)s += 5 후return s
1번째055
2번째51010
3번째101515
4번째 (printf)152020
⚠️ 일반 지역변수 vs static: 일반이면 매 호출마다 s=0으로 초기화 → 항상 5 출력. static이면 이전 값 유지 → 5, 10, 15, 20 누적.

07 문자열 포인터
문자열 리터럴은 메모리에 연속 저장됨. char *p = "KOREA" 에서 p는 'K'의 주소.
%s: p부터 '\0'까지 출력  |  %c: 단일 문자 출력

문자열 포인터 추적 — 2023 3회 Q10 유사

1#include <stdio.h>
2int main() {
3    char *p = "KOREA";   // K O R E A \0
4                          // 0 1 2 3 4  5
5    printf("%s\n", p);     // p+0 부터 → KOREA
6    printf("%s\n", p+1);   // p+1 부터 → OREA
7    printf("%c\n", *p);    // p[0] → K
8    printf("%c\n", *(p+3)); // p[3] → E
9    printf("%c\n", *(p+1)); // p[1] → O
10   return 0;
11}
라인포맷시작 위치출력
5%sp+0 = 'K'KOREA
6%sp+1 = 'O'OREA
7%c*p = p[0]K
8%c*(p+3) = p[3]E
9%c*(p+1) = p[1]O

2024 2회 Q18  다음 C코드의 출력 결과를 쓰시오.
▼ 
#include <stdio.h>
void swap(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
}
int main() {
    int a = 5, b = -13;
    swap(a, b);
    switch (a) {
        case  5: printf("%d", b); break;
        case -13: printf("%d", a); break;
    }
    return 0;
}
정답
-13
Call by Value: swap()에 a,b의 복사본이 전달됨. 함수 내부에서 temp를 이용한 교환은 지역 복사본에만 영향 → main의 a, b는 그대로 5, -13.
switch(a=5): case 5 → printf(b) → -13
2024 1회 Q2 유사  다음 C코드의 출력 결과를 쓰시오. (삼항연산자 + 비트 연산)
▼ 
#include <stdio.h>
int main() {
    int a = 9, b = 4;
    int c = (a > b) ? a : b;  // 삼항
    c = (c << b) | 7;         // 비트
    printf("%d", c);
    return 0;
}
정답
151
a > b (9 > 4) → true → c = a = 9
c << 4 = 9 × 16 = 144 (10010000₂)
144 | 7 = 10010000 | 00000111 = 10010111₂ = 151
2024 3회 Q16 유사  다음 C코드의 출력 결과를 쓰시오. (이중 포인터)
▼ 
#include <stdio.h>
int main() {
    int x = 4;
    int  *p  = &x;   // p  → x
    int **pp = &p;   // pp → p → x
    **pp = **pp / x;  // x = x / x = 4/4 = 1
    printf("%d", x);
    return 0;
}
정답
1
**pp는 pp → p → x, 즉 x 자체를 참조.
**pp = **pp / x → x = 4 / 4 = 1
printf(x) → 1

2023 3회 Q4 유사  다음 C코드의 출력 결과를 쓰시오. (완전수 합)
▼ 
#include <stdio.h>
int main() {
    int sum = 0;
    for (int n = 1; n <= 100; n++) {
        int dsum = 0;
        for (int i = 1; i < n; i++)
            if (n % i == 0) dsum += i;
        if (dsum == n) sum += n;  // 완전수
    }
    printf("%d", sum);
    return 0;
}
정답
34
완전수: 자기 자신을 제외한 약수의 합 = 자기 자신인 수
6 = 1+2+3 ✅    28 = 1+2+4+7+14 ✅
1~100 사이 완전수: 6, 28 → 합 = 34
기출 유사  다음 C코드의 출력 결과를 쓰시오. (포인터 + 비트 연산 혼합)
▼ 
#include <stdio.h>
int main() {
    int a = 12, b = 10;
    int *p = &a;
    printf("%d\n", *p & b);    // AND
    printf("%d\n", *p | b);    // OR
    printf("%d\n", *p ^ b);    // XOR
    printf("%d\n", *p >> 1);  // Right Shift
    return 0;
}
정답
8
14
6
6
*p = a = 12 = 1100₂, b = 10 = 1010₂
12 & 10 = 1100 & 1010 = 1000 = 8
12 | 10 = 1100 | 1010 = 1110 = 14
12 ^ 10 = 1100 ^ 1010 = 0110 = 6
12 >> 1 = 0110 = 6 (÷2)
개념 확인  Call by Value와 Call by Reference의 차이를 설명하고, C에서 참조 전달을 구현하는 방법을 쓰시오.
▼
정답
포인터를 매개변수로 전달
Call by Value: 값의 복사본 전달 → 함수 내 변경이 원본에 영향 없음
Call by Reference: C에는 없음 → 포인터로 구현: void swap(int *a, int *b)
포인터를 받아 *a, *b로 역참조하면 원본 수정 가능.
