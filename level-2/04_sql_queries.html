05 서브쿼리
유형특징연산자
단일행 서브쿼리서브쿼리 결과가 1행 1열= < > <= >=
다중행 서브쿼리서브쿼리 결과가 여러 행IN, NOT IN, ANY, ALL, EXISTS
상관 서브쿼리외부 쿼리 값을 서브쿼리가 참조 (행마다 실행)EXISTS, IN
스칼라 서브쿼리SELECT 절에 위치, 결과 1행 1열 반드시—

단일행 서브쿼리 — 최고 급여 사원

SELECT NAME, SALARY
FROM EMP
WHERE SALARY = (SELECT MAX(SALARY) FROM EMP);
-- 서브쿼리 결과: 5000
-- 결과: 최현아, 5000

다중행 서브쿼리 — IN / NOT IN

-- 서울에 있는 부서의 사원만
SELECT NAME, DID
FROM EMP
WHERE DID IN (
    SELECT DID FROM DEPT WHERE LOC = '서울'
);
-- 서브쿼리 결과: ('D1', 'D4')
-- EMP 중 DID IN ('D1','D4'): 김민수,박준영,한지수 (D4 해당 없음)
✅ ANY: 하나라도 참이면 TRUE | ALL: 모두 참이어야 TRUE
> ANY(3500, 4200, 5000) → 3500보다 크면 OK (= > MIN)
> ALL(3500, 4200, 5000) → 5000보다도 크면 OK (= > MAX)

상관 서브쿼리 — 소속 부서 평균 이상 받는 사원

SELECT e1.NAME, e1.DID, e1.SALARY
FROM EMP e1
WHERE e1.SALARY > (
    SELECT AVG(e2.SALARY)
    FROM EMP e2
    WHERE e2.DID = e1.DID  -- 외부 쿼리의 e1 참조
);
-- D1 평균: (3500+3800+4100)/3 = 3800 → 한지수(4100) 해당
-- D2 평균: (4200+3200)/2 = 3700 → 이지은(4200) 해당
-- D3 평균: 5000 → 없음
NAMEDIDSALARY부서평균
이지은D242003700
한지수D141003800

스칼라 서브쿼리 — SELECT 절에서 사용

SELECT
    e.NAME,
    e.SALARY,
    (SELECT d.DNAME FROM DEPT d WHERE d.DID = e.DID) AS DNAME,
    e.SALARY - (SELECT AVG(SALARY) FROM EMP) AS DIFF
FROM EMP e;
-- 전체 평균: (3500+4200+3800+5000+3200+4100)/6 ≈ 3966.7

관계대수 → SQL 변환 예제

-- 관계대수: π_NAME,SALARY(σ_SALARY≥4000(EMP))
-- 읽기: EMP에서 SALARY≥4000인 행 선택 후 NAME, SALARY만 추출

SELECT NAME, SALARY
FROM EMP

-- 관계대수: π_NAME,DNAME(EMP ⋈ DEPT)

07 DDL — CREATE / ALTER / DROP

CREATE TABLE — 테이블 생성

CREATE TABLE PRODUCT (
    PID       INT          PRIMARY KEY,          -- 기본키
    PNAME     VARCHAR(50) NOT NULL,              -- NULL 불가
    PRICE     INT          DEFAULT 0,             -- 기본값 0
    CID       INT          UNIQUE,                -- 중복 불가
    STOCK     INT          CHECK (STOCK >= 0),   -- 체크 제약
    DEPT_ID   VARCHAR(10) REFERENCES DEPT(DID)  -- 외래키
);
제약조건의미
PRIMARY KEY기본키: NOT NULL + UNIQUE 조합, 테이블당 1개
FOREIGN KEY외래키: 다른 테이블 기본키 참조, 참조 무결성 보장
UNIQUE유일성: NULL은 여러 개 허용 (구현마다 다름)
NOT NULLNULL 입력 불가
CHECK지정 조건을 만족해야 입력 허용
DEFAULTINSERT 시 값 생략하면 기본값 자동 삽입

ALTER / DROP

-- 열 추가
ALTER TABLE EMP ADD EMAIL VARCHAR(100);

-- 열 수정 (데이터타입 변경)
ALTER TABLE EMP MODIFY NAME VARCHAR(100);

-- 열 삭제
ALTER TABLE EMP DROP COLUMN EMAIL;

-- 테이블 삭제 (데이터 + 구조 모두)
DROP TABLE EMP;

-- 데이터만 삭제 (구조 유지)
TRUNCATE TABLE EMP;

-- 특정 데이터 삭제 (롤백 가능)
DELETE FROM EMP WHERE DID = 'D3';
⚠️ DROP ≠ TRUNCATE ≠ DELETE
DROP: 테이블 자체 삭제 (롤백 불가) | TRUNCATE: 데이터만 전체 삭제 (롤백 불가) | DELETE: 조건부 삭제 (롤백 가능)
08 기출 문제
기출 빈출  다음 SQL의 결과 행 수를 쓰시오. (JOIN 이해)
▼ 
-- 위 예제 테이블 기준 (EMP 6행, DEPT 3행)

-- ① INNER JOIN
SELECT * FROM EMP e INNER JOIN DEPT d ON e.DID = d.DID;

-- ② LEFT JOIN
SELECT * FROM EMP e LEFT JOIN DEPT d ON e.DID = d.DID;

-- ③ RIGHT JOIN
SELECT * FROM EMP e RIGHT JOIN DEPT d ON e.DID = d.DID;
정답
① 5행   ② 6행   ③ 6행
① INNER: D1(김민수,박준영,한지수=3) + D2(이지은,정세훈=2) = 5행 (최현아 D3 미매칭, D4 사원 없음)
② LEFT: EMP 6행 전부 + 최현아 DNAME=NULL = 6행
③ RIGHT: D1(3행)+D2(2행)+D4(NULL,기획팀,NULL) = 6행
기출 빈출  다음 SQL의 출력 결과를 쓰시오. (GROUP BY + HAVING)
▼ 
SELECT DID, COUNT(*) AS CNT, SUM(SALARY) AS TOTAL
FROM EMP
WHERE SALARY < 5000
GROUP BY DID
HAVING COUNT(*) >= 2
ORDER BY TOTAL DESC;
정답
D1, 3, 11400
D2, 2, 7400
WHERE SALARY < 5000 → 최현아(5000) 제외
D1: 3500+3800+4100=11400, D2: 4200+3200=7400
HAVING COUNT≥2 → D1(3), D2(2) 통과
ORDER BY TOTAL DESC → D1(11400) 먼저
기출 빈출  다음 SQL에서 빈칸 (㉠), (㉡)에 알맞은 내용을 쓰시오.
급여가 전체 평균 이상인 사원의 이름과 급여를 조회하는 쿼리
▼ 
SELECT NAME, SALARY
FROM EMP
WHERE SALARY ( ㉠ ) (
    SELECT ( ㉡ )(SALARY) FROM EMP
);
정답
㉠ >=    ㉡ AVG
전체 평균 이상 → SALARY >= AVG(SALARY)
AVG(SALARY) = (3500+4200+3800+5000+3200+4100)/6 ≈ 3966.7
결과: 이지은(4200), 최현아(5000), 한지수(4100)
기출 빈출  다음 SQL의 결과로 출력되는 행 수를 쓰시오. (서브쿼리 NOT IN + NULL 주의)
▼ 
-- DEPT2 테이블 (DID: D1, D2, NULL)
SELECT NAME FROM EMP
WHERE DID NOT IN (
    SELECT DID FROM DEPT2
    -- 결과: ('D1', 'D2', NULL)
);
정답
0행 (결과 없음)
🚨 NOT IN + NULL 함정: 서브쿼리 결과에 NULL이 포함되면 NOT IN 조건이 항상 UNKNOWN이 되어 아무 행도 반환하지 않음.
→ NULL과의 비교는 항상 UNKNOWN (TRUE도 FALSE도 아님)
→ 해결: WHERE DID NOT IN (SELECT DID FROM DEPT2 WHERE DID IS NOT NULL)
기출 빈출  다음 관계대수식을 SQL로 변환하시오.
▼
πNAME, SALARY(σDID='D1' ∧ SALARY≥3800(EMP))
정답
SELECT NAME, SALARY
FROM EMP
WHERE DID = 'D1' AND SALARY >= 3800;
σDID='D1' ∧ SALARY≥3800 → WHERE DID='D1' AND SALARY>=3800 (Selection)
πNAME, SALARY → SELECT NAME, SALARY (Projection)

기출 빈출  다음 SQL의 출력 결과를 쓰시오. (UNION vs UNION ALL)
▼ 
-- A 테이블: (1,'김'), (2,'이'), (3,'박')
-- B 테이블: (2,'이'), (4,'최'), (5,'정')

-- ① UNION (중복 제거)
SELECT * FROM A UNION SELECT * FROM B;

-- ② UNION ALL (중복 포함)
SELECT * FROM A UNION ALL SELECT * FROM B;
정답
① 5행  (중복 제거)
② 6행  (중복 포함)
A∪B = {1,2,3,4,5} → 5행 (2,'이' 중복 1개 제거)
UNION ALL: A(3행) + B(3행) = 6행 (2,'이' 2번 포함)
기출 유사  다음 SQL에서 빈칸을 채워 2번째로 높은 급여를 구하는 쿼리를 완성하시오.
▼ 
SELECT MAX(SALARY)
FROM EMP
WHERE SALARY < (
    SELECT ( 빈칸 )(SALARY) FROM EMP
);
정답
MAX
서브쿼리로 MAX(SALARY)=5000을 구하고, 그보다 작은 것들 중 MAX → 4200
WHERE SALARY < 5000 → {3500,4200,3800,3200,4100} → MAX = 4200
개념 확인  WHERE 절과 HAVING 절의 차이를 설명하고, 다음 중 오류가 있는 쿼리를 고르시오.
▼ 
-- ① (정상)
SELECT DID, AVG(SALARY) FROM EMP
GROUP BY DID HAVING AVG(SALARY) > 3000;

-- ② (오류)
SELECT DID, AVG(SALARY) FROM EMP
WHERE AVG(SALARY) > 3000
GROUP BY DID;
정답
② 오류
WHERE 절에는 집계함수 사용 불가: WHERE는 GROUP BY 전에 실행되므로 아직 그룹이 없음
집계함수(AVG, COUNT, SUM 등) 조건은 반드시 HAVING 절에 작성
① 정상: GROUP BY → AVG 계산 → HAVING으로 필터 (올바른 순서)
